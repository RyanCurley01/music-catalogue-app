## Overview

The Music Catalogue App allows users to create, view, edit, and delete music tracks. Each track stores a title, album, artist, genre, release date, and rating. The app includes several discovery features — **Find Music by Same Artist**, **Find Music by Same Genre**, **Filter by Rating**, and **Sort by Column Headings** — making it easy to explore the catalogue in multiple ways.

---

## Features

### Core CRUD
- **List** all music tracks on the home page
- **View** full details of any track
- **Add** new tracks via a form
- **Edit** existing track information (title, album, artist, genre, release date, rating)
- **Delete** tracks from the catalogue

### Same Artist Lookup
From any track's detail page, users can click **"Find Music With Same Artist"** to see all other tracks in the catalogue by the same artist. If no artist information is available for a track, the user is redirected to the home page with an informative flash message.

### Same Genre Lookup
From any track's detail page, users can click **"Find Music With Same Genre"** to see all other tracks in the catalogue that share the same genre. If no genre information is available for a track, the user is redirected to the home page with an informative flash message.

### Filter by Rating
On the home page, users can filter the catalogue by a star rating (1–5) using a dropdown and a **Search** button. Only tracks matching the selected rating are displayed.

### Sort by Column Headings
On the home page, all column headings — **Title**, **Album**, **Artist**, **Genre**, **Release Date**, and **Rating** — are clickable links that sort the catalogue by that field. Sorting preserves any active rating filter.

## Getting Started

### Prerequisites

- Ruby 3.3.8
- Bundler
- SQLite3

### Setup

```bash
# Clone the repository
git clone https://github.com/RyanCurley01/music-catalogue-app.git
cd music-catalogue-app

# Install dependencies
bundle install

# Set up and seed the database
bundle exec rails db:migrate
bundle exec rails db:seed

# Start the development server
rails server -b 0.0.0.0
```

Visit `http://localhost:3000` in your browser.

---

## Running Tests

Always run tests using `bundle exec` to ensure the correct gem environment is used.

```bash
# Run Cucumber BDD acceptance tests
bundle exec cucumber

# Run RSpec unit and functional tests
bundle exec rspec

# Prepare the test database (if needed before running tests)
bundle exec rails db:test:prepare
```

Code coverage reports are automatically generated by SimpleCov and saved to `coverage/index.html` after each test run.

## Routes

| Method | Path | Action | Description |
|---|---|---|---|
| GET | `/` | `music#index` | List all tracks (supports `?sort=` and `?rating=` params) |
| GET | `/music/new` | `music#new` | New track form |
| POST | `/music` | `music#create` | Create a track |
| GET | `/music/:id` | `music#show` | Track details |
| GET | `/music/:id/edit` | `music#edit` | Edit form |
| PATCH | `/music/:id` | `music#update` | Update a track |
| DELETE | `/music/:id` | `music#destroy` | Delete a track |
| GET | `/music/:id/same_artist` | `music#same_artist` | Find tracks by same artist |
| GET | `/music/:id/same_genre` | `music#same_genre` | Find tracks by same genre |

## User Stories

### Same Artist Lookup

> **As an** electronic music fan,  
> **So that** I can find music with my favourite artist,  
> **I want to** search for and view other tracks in the catalogue by the same artist.

Covered by Cucumber tests in `features/filter_by_artist.feature`, including happy path (results displayed) and sad path (no artist info, redirect with notice) scenarios.

### Same Genre Lookup

> **As an** electronic music fan,  
> **So that** I can find styles of music I like,  
> **I want to** filter and search music by the same genre.

Covered by Cucumber tests in `features/filter_by_genre.feature`, including happy path (results displayed) and sad path (no genre info, redirect with notice) scenarios.

### Filter by Rating

> **As an** electronic music fan,  
> **So that** I can save styles of music I like,  
> **I want to** add a rating from 1 to 5 and filter the catalogue by rating.

Covered by Cucumber tests in `features/filter_by_rating.feature`, including filtering to matching tracks and confirming non-matching tracks are hidden.

### Sort by Column Headings

> **As an** electronic music fan,  
> **So that** I can browse the catalogue in the order I prefer,  
> **I want to** sort tracks by clicking any column heading.

Covered by Cucumber tests in `features/sort_by_headings.feature`, with scenarios for sorting by title, album, artist, genre, release date, and rating.
